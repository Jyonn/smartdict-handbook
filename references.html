Feature Page

References

smartdict's core behavior is reference resolution. This page explains how string interpolation, native-value preservation, nested references, key generation, and object-backed lookups behave in practice.

Case 1

Interpolation inside a larger string

python 
smartdict.parse({
    "name": "smartdict",
    "message": "hello-${name}",
})
# {'name': 'smartdict', 'message': 'hello-smartdict'}

When a reference appears inside a larger string, smartdict always returns a string.

Case 2

Single references preserve native Python values

Default syntax

smartdict.parse({
    "config": {"debug": True},
    "selected": "${config}",
})
# {'config': {'debug': True}, 'selected': {'debug': True}}

Explicit full-match form

smartdict.parse({
    "config": {"debug": True},
    "selected": "${config}$",
})
# {'config': {'debug': True}, 'selected': {'debug': True}}
Takeaway: ${...} is the main syntax. ${...}$ is still valid, but now mainly serves as an explicit full-match marker for human readers.

Case 3

Nested reference strings

python 
smartdict.parse({
    "env": "prod",
    "keys": {"prod": "url"},
    "url": "https://example.com",
    "result": "${${keys.${env}}}",
})
# {'env': 'prod', 'keys': {'prod': 'url'}, 'url': 'https://example.com', 'result': 'https://example.com'}

smartdict can first resolve the inner expression, then use the result as the path for the outer expression.

Case 4

Keys can be generated too

smartdict.parse({
    "name": "k",
    "${name}": 1,
})
# {'name': 'k', 'k': 1}

smartdict resolves references in both dictionary keys and dictionary values.

Case 5

Lists, tuples, and nested paths

smartdict.parse({
    "items": ["a", "b"],
    "pair": ("x", "y"),
    "pick_list": "${items.1}",
    "pick_tuple": "${pair.0}",
})
# {'items': ['a', 'b'], 'pair': ('x', 'y'), 'pick_list': 'b', 'pick_tuple': 'x'}

Case 6

Object attributes and mixed object/dict paths

Attribute access

class A:
    def __init__(self):
        self.x = "value"

smartdict.parse({
    "a": A(),
    "b": "${a.x}",
})
# {'a': <A ...>, 'b': 'value'}

Mixed paths

class Config:
    def __init__(self):
        self.profile = "prod"

smartdict.parse({
    "app": Config(),
    "services": {"prod": {"url": "https://example.com"}},
    "result": "${services.${app.profile}.url}",
})
# {'app': <Config ...>, 'services': {'prod': {'url': 'https://example.com'}}, 'result': 'https://example.com'}
Lookup order: for each path component smartdict tries obj[key], then getattr(obj, key), then obj[int(key)].
