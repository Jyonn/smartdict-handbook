Feature Page

Modes & errors

smartdict gives you three parsing strategies and several focused exception types so configs can either fail fast or stay partially unresolved, depending on the phase of your workflow.

Mode 1

smartdict.parse(obj)

Strict mode. Resolves everything it can and raises an error if unresolved references remain at the end. 

smartdict.parse({
    "a": "${missing}",
})
# raises ReferenceNotFoundError

Mode 2

smartdict.partial_parse(obj)

Best-effort mode. Missing references stay as placeholders such as ${missing} instead of raising. 

smartdict.partial_parse({
    "a": "${missing}",
    "b": "pre-${missing}-post",
    "c": "${missing}$",
})
# {
#   'a': '${missing}',
#   'b': 'pre-${missing}-post',
#   'c': '${missing}$'
# }

Mode 3

smartdict.iterative_parse(obj, iterations=N)

Repeated best-effort parsing. This is useful when the first pass resolves values that unlock later passes. 

smartdict.iterative_parse({
    "a": "${b}",
    "b": "${c}",
    "c": "ok",
}, iterations=2)
# {'a': 'ok', 'b': 'ok', 'c': 'ok'}
Guardrail: iterations must be greater than zero. Passing 0 raises ValueError.

Errors

What each exception means

ReferenceNotFoundError

Raised in strict mode when unresolved references remain. The exception exposes structured entries in exception.unresolved.

CircularReferenceError

Raised when a dependency chain loops back to itself, including cycles across nested dictionaries.

PipelineStageError

Raised when a pipeline stage is unknown or cannot convert the current value, such as ${env.PORT:abc|int}.

ValueError

Raised for invalid parser configuration such as iterations <= 0.

Examples

Error-shaped scenarios worth knowing

Nested missing reference

smartdict.parse({
    "app": {"profile": "prod"},
    "services": {"prod": {"url": "${config.endpoints.api}"}},
    "result": "${services.${app.profile}.url}",
})
# raises ReferenceNotFoundError

Cross-nested cycle

smartdict.parse({
    "app": {"profile": "${services.primary.profile}$"},
    "services": {"primary": {"profile": "${app.profile}$"}},
})
# raises CircularReferenceError
