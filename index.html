smartdict 0.5.0

Typed config references.

smartdict is a small Python library for resolving references inside nested dictionaries, lists, tuples, and object-backed configs. It supports native value preservation, JSON defaults, lightweight pipelines, and strict or partial parsing modes.

Learn the Syntax Open Playground
quick-start.py 
import smartdict

config = {
    "dataset": "Spotify Search",
    "save_dir": "${dataset|strip|lower|slug}",
    "port": "${env.PORT:8000|int}",
    "metadata": '${details:{"owner": "jyonn"}}',
}

print(smartdict.parse(config))
# {
#   'dataset': 'Spotify Search',
#   'save_dir': 'spotify-search',
#   'port': 8000,
#   'metadata': {'owner': 'jyonn'}
# }

Quick Start

Minimal config example

python 
import smartdict

data = {
    "dataset": "spotify",
    "load": {
        "base_path": "~/data/${dataset}",
        "train_path": "${load.base_path}/train",
        "dev_path": "${load.base_path}/dev",
        "test_path": "${load.base_path}/test",
    },
    "network": {
        "num_hidden_layers": 3,
        "num_attention_heads": 8,
    },
    "store": "checkpoints/${dataset}/${network.num_hidden_layers}L${network.num_attention_heads}H/",
}

parsed = smartdict.parse(data)

print(parsed["load"]["base_path"])
# ~/data/spotify

print(parsed["load"]["dev_path"])
# ~/data/spotify/dev

print(parsed["store"])
# checkpoints/spotify/3L8H/

Scenarios

Common ways people use smartdict

ML experiment configs

Reuse dataset names, checkpoint paths, and model fallback chains without scattering string formatting through training code.

Service runtime settings

Parse ports, booleans, or JSON payloads with typed defaults such as ${env.PORT:8000|int}.

Structured metadata

Attach typed object defaults like ${config:{"owner": "jyonn"}} without post-processing.

Hybrid object + dict configs

Resolve values from custom Python objects, plain dictionaries, lists, and tuples within the same config tree.
