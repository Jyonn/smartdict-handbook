Feature Page

Pipelines transform values after resolution

Pipeline syntax follows ${ref[:default][|stage]...}. SmartDict resolves the reference first, applies the default if missing, then runs stages from left to right.

Execution Model

The order is fixed

1. Resolve

Look up the reference from the current config object.

2. Default

Use the fallback value if the reference is unresolved.

3. Transform

Apply stages left to right, preserving native values where possible.

Case 1

String cleanup and slug generation

python 
smartdict.parse({
    "dataset": "  My Dataset  ",
    "save_dir": "${dataset|strip|lower|slug}",
})
# {'dataset': '  My Dataset  ', 'save_dir': 'my-dataset'}

Case 2

Typed fallback values followed by conversion

Fallback path

smartdict.parse({
    "port": "${env.PORT:8000|int}",
})
# {'port': 8000}

Existing value path

smartdict.parse({
    "env": {"PORT": "9000"},
    "port": "${env.PORT:8000|int}",
})
# {'env': {'PORT': '9000'}, 'port': 9000}

Case 3

JSON stage

smartdict.parse({
    "raw": '{"hello": "world"}',
    "value": "${raw|json}",
})
# {'raw': '{"hello": "world"}', 'value': {'hello': 'world'}}

Case 4

Pipelines can follow nested defaults

smartdict.parse({
    "repr_source_model": "TEXT-EMBEDDING-3-SMALL",
    "embedding_model": "${sid_embedding_model:${repr_source_model:null}|lower}",
})
# {'repr_source_model': 'TEXT-EMBEDDING-3-SMALL', 'embedding_model': 'text-embedding-3-small'}

Failure Modes

Pipeline stage errors are structured

smartdict.parse({
    "port": "${env.PORT:abc|int}",
})
# raises PipelineStageError

Unknown stages and failed conversions both raise PipelineStageError, which includes the stage name, failing value, and field path.
