Feature Page

Defaults & JSON

smartdict treats defaults as typed data whenever possible. Scalars, lists, dictionaries, nested fallback chains, and JSON values can all survive parsing without being collapsed into strings.

Case 1

Scalar defaults

python 
smartdict.parse({
    "int_value": "${missing:42}",
    "bool_value": "${missing:true}",
    "null_value": "${missing:null}",
    "text_value": "${missing:fallback}",
})
# {
#   'int_value': 42,
#   'bool_value': True,
#   'null_value': None,
#   'text_value': 'fallback'
# }
Rule: smartdict first tries JSON parsing for defaults, then falls back to the original loose string behavior so bare strings like fallback still work.

Case 2

JSON array and object defaults

List default

smartdict.parse({
    "sinkhorn_epsilon": "${sid_sinkhorn_epsilon:[0.0, 0.0, 0.003]}",
})
# {'sinkhorn_epsilon': [0.0, 0.0, 0.003]}

Dict default

smartdict.parse({
    "metadata": '${config:{"hello": "world"}}',
})
# {'metadata': {'hello': 'world'}}

Case 3

JSON values can contain colons safely

smartdict.parse({
    "value": '${config:{"hello": "a:b", "url": "https://example.com"}}',
})
# {'value': {'hello': 'a:b', 'url': 'https://example.com'}}

The parser distinguishes the top-level default separator from the colons inside the JSON object itself.

Case 4

Primary value wins over the default

smartdict.parse({
    "sid_sinkhorn_epsilon": [1.0, 2.0, 3.0],
    "sinkhorn_epsilon": "${sid_sinkhorn_epsilon:[0.0, 0.0, 0.003]}",
})
# {'sid_sinkhorn_epsilon': [1.0, 2.0, 3.0], 'sinkhorn_epsilon': [1.0, 2.0, 3.0]}

Case 5

Nested fallback chains

python 
smartdict.parse({
    "repr_source_model": "text-embedding-3-small",
    "embedding_model": "${sid_embedding_model:${repr_source_model:null}}",
})
# {'repr_source_model': 'text-embedding-3-small', 'embedding_model': 'text-embedding-3-small'}

smartdict.parse({
    "embedding_model": "${sid_embedding_model:${repr_source_model:null}}",
})
# {'embedding_model': None}

You can chain fallbacks so one missing value falls back to another reference, which may itself have a typed default.
